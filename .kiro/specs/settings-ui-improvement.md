# 설정 UI 개선 사양서 (Settings UI Improvement Specification)

## 프로젝트 개요
Fazzk 팔로워 알림 앱의 설정 인터페이스를 더 직관적이고 사용자 친화적으로 개선하는 프로젝트입니다.

## 현재 상태 분석

### 기존 설정 UI 문제점
1. **단일 모달 구조**: 모든 설정이 하나의 긴 모달에 나열되어 있어 복잡함
2. **카테고리 부족**: 관련 설정들이 논리적으로 그룹화되지 않음
3. **시각적 피드백 부족**: 설정 변경 시 실시간 미리보기가 없음
4. **접근성 문제**: 키보드 네비게이션과 스크린 리더 지원 부족
5. **모바일 친화성 부족**: 작은 화면에서 사용하기 어려움
6. **설정 검색 기능 없음**: 많은 설정 중 원하는 것을 찾기 어려움

### 현재 설정 항목들
- **오디오 설정**: 볼륨, 커스텀 사운드, TTS
- **알림 설정**: 갱신 주기, 표시 시간, 레이아웃, 애니메이션
- **외관 설정**: 텍스트 색상, 텍스트 크기
- **OBS 연동**: URL 복사, 권장 크기 안내

## 개선 목표

### 주요 목표
1. **직관적인 네비게이션**: 탭 기반 또는 사이드바 네비게이션으로 설정 카테고리 분리
2. **실시간 미리보기**: 설정 변경 시 즉시 결과를 확인할 수 있는 프리뷰 기능
3. **향상된 접근성**: WCAG 2.1 AA 수준의 접근성 준수
4. **반응형 디자인**: 다양한 화면 크기에서 최적화된 경험
5. **설정 검색**: 빠른 설정 찾기 기능
6. **설정 프리셋**: 일반적인 사용 시나리오별 미리 구성된 설정

### 성능 목표
- 설정 모달 로딩 시간: 100ms 이하
- 설정 변경 반영 시간: 50ms 이하
- 메모리 사용량 증가: 5MB 이하

## 사용자 스토리

### Epic 1: 카테고리별 설정 관리
**As a** 스트리머
**I want** 설정을 논리적 카테고리로 나누어 관리하고 싶다
**So that** 원하는 설정을 빠르게 찾고 수정할 수 있다

#### User Story 1.1: 탭 기반 네비게이션
- **Given** 설정 모달이 열려있을 때
- **When** 사용자가 다른 카테고리 탭을 클릭하면
- **Then** 해당 카테고리의 설정들만 표시되어야 한다

#### User Story 1.2: 설정 카테고리 구성
- **오디오**: 볼륨, 사운드 파일, TTS 설정
- **알림**: 갱신 주기, 표시 시간, 애니메이션
- **외관**: 레이아웃, 색상, 크기, 테마
- **연동**: OBS 설정, 키보드 단축키
- **고급**: 성능 설정, 디버그 옵션

### Epic 2: 실시간 미리보기
**As a** 사용자
**I want** 설정을 변경할 때 실시간으로 결과를 미리 볼 수 있다
**So that** 원하는 결과를 얻을 때까지 시행착오를 줄일 수 있다

#### User Story 2.1: 알림 미리보기
- **Given** 알림 설정을 변경할 때
- **When** 사용자가 슬라이더나 드롭다운을 조작하면
- **Then** 설정 모달 내에서 알림 샘플이 실시간으로 업데이트되어야 한다

#### User Story 2.2: 오디오 미리보기
- **Given** 볼륨이나 사운드 설정을 변경할 때
- **When** 사용자가 설정을 조작하면
- **Then** 즉시 테스트 사운드가 재생되어야 한다

### Epic 3: 향상된 사용성
**As a** 사용자
**I want** 더 편리하고 접근 가능한 설정 인터페이스를 사용하고 싶다
**So that** 장애가 있거나 다양한 입력 방식을 사용해도 쉽게 설정할 수 있다

#### User Story 3.1: 키보드 네비게이션
- **Given** 설정 모달이 열려있을 때
- **When** 사용자가 Tab 키를 누르면
- **Then** 논리적 순서로 포커스가 이동해야 한다

#### User Story 3.2: 설정 검색
- **Given** 설정 모달 상단에 검색 박스가 있을 때
- **When** 사용자가 키워드를 입력하면
- **Then** 관련 설정들만 필터링되어 표시되어야 한다

### Epic 4: 설정 프리셋
**As a** 초보 사용자
**I want** 미리 구성된 설정 프리셋을 사용하고 싶다
**So that** 복잡한 설정을 일일이 조정하지 않고도 좋은 결과를 얻을 수 있다

#### User Story 4.1: 프리셋 선택
- **Given** 설정 모달에 프리셋 섹션이 있을 때
- **When** 사용자가 "스트리밍 최적화", "조용한 알림", "화려한 효과" 등의 프리셋을 선택하면
- **Then** 관련된 모든 설정이 자동으로 적용되어야 한다

## 기술 사양

### 아키텍처 설계
```
SettingsModal.svelte (메인 컨테이너)
├── SettingsNavigation.svelte (탭/사이드바)
├── SettingsSearch.svelte (검색 기능)
├── SettingsPresets.svelte (프리셋 관리)
├── AudioSettings.svelte (오디오 설정)
├── NotificationSettings.svelte (알림 설정)
├── AppearanceSettings.svelte (외관 설정)
├── IntegrationSettings.svelte (연동 설정)
├── AdvancedSettings.svelte (고급 설정)
└── SettingsPreview.svelte (실시간 미리보기)
```

### 상태 관리
```typescript
interface SettingsState {
  activeCategory: string;
  searchQuery: string;
  previewMode: boolean;
  isDirty: boolean;
  presets: SettingsPreset[];
}

interface SettingsPreset {
  id: string;
  name: string;
  description: string;
  settings: Partial<AppSettings>;
}
```

### API 확장
```rust
// src-tauri/src/server.rs에 추가할 엔드포인트
GET /settings/presets - 프리셋 목록 조회
POST /settings/presets - 새 프리셋 생성
PUT /settings/presets/{id} - 프리셋 수정
DELETE /settings/presets/{id} - 프리셋 삭제
POST /settings/preview - 설정 미리보기 (임시 적용)
```

## 구현 계획

### Phase 1: 기본 구조 개선 (1-2일)
1. 현재 설정 모달을 컴포넌트로 분리
2. 탭 기반 네비게이션 구현
3. 카테고리별 설정 그룹화
4. 기본 반응형 레이아웃 적용

### Phase 2: 미리보기 기능 (2-3일)
1. 설정 미리보기 컴포넌트 개발
2. 실시간 알림 샘플 표시
3. 오디오 테스트 기능 구현
4. 외관 변경 즉시 반영

### Phase 3: 접근성 및 사용성 (1-2일)
1. 키보드 네비게이션 구현
2. ARIA 라벨 및 역할 추가
3. 설정 검색 기능 개발
4. 포커스 관리 개선

### Phase 4: 프리셋 시스템 (2-3일)
1. 프리셋 데이터 구조 설계
2. 백엔드 API 개발
3. 프리셋 UI 구현
4. 기본 프리셋 제공

### Phase 5: 최적화 및 테스트 (1일)
1. 성능 최적화
2. 접근성 테스트
3. 다양한 화면 크기 테스트
4. 사용자 피드백 반영

## 수용 기준

### 기능적 요구사항
- [ ] 5개 카테고리로 설정이 논리적으로 분류됨
- [ ] 탭 클릭으로 카테고리 간 전환 가능
- [ ] 설정 변경 시 실시간 미리보기 제공
- [ ] 검색어 입력으로 관련 설정 필터링 가능
- [ ] 최소 3개의 기본 프리셋 제공
- [ ] 모든 설정이 정상적으로 저장/로드됨

### 비기능적 요구사항
- [ ] 설정 모달 로딩 시간 100ms 이하
- [ ] 키보드만으로 모든 기능 접근 가능
- [ ] 320px 이상의 화면에서 정상 작동
- [ ] 색상 대비비 4.5:1 이상 유지
- [ ] 메모리 사용량 5MB 이하 증가

### 호환성 요구사항
- [ ] 기존 설정 데이터 완전 호환
- [ ] OBS 브라우저 소스에서 정상 작동
- [ ] Tauri 앱과 브라우저 모드 모두 지원

## 위험 요소 및 대응 방안

### 기술적 위험
1. **복잡성 증가**: 컴포넌트 분리로 인한 상태 관리 복잡성
   - **대응**: Svelte의 reactive statements와 stores 활용
2. **성능 저하**: 실시간 미리보기로 인한 성능 영향
   - **대응**: 디바운싱과 가상화 기법 적용
3. **호환성 문제**: 기존 설정 데이터와의 호환성
   - **대응**: 마이그레이션 스크립트 및 fallback 로직 구현

### 사용자 경험 위험
1. **학습 곡선**: 새로운 UI에 대한 사용자 적응
   - **대응**: 점진적 개선과 기존 UI 옵션 제공
2. **접근성 문제**: 복잡한 UI로 인한 접근성 저하
   - **대응**: 접근성 테스트 도구 활용 및 사용자 피드백 수집

## 성공 지표

### 정량적 지표
- 설정 변경 완료율: 95% 이상
- 설정 모달 이탈률: 10% 이하
- 평균 설정 완료 시간: 30% 단축
- 접근성 점수: WCAG 2.1 AA 수준 달성

### 정성적 지표
- 사용자 만족도 조사: 4.5/5.0 이상
- 설정 관련 문의 감소: 50% 이상
- 새 사용자 온보딩 성공률 향상
