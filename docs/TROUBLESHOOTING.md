# Fazzk 트러블슈팅 가이드

## 일반적인 문제 해결

### 1. 로그인 관련 문제

#### 문제: "로그인에 실패했습니다" 오류
**원인:**
- 쿠키가 만료됨
- 잘못된 쿠키 형식
- 네트워크 연결 문제

**해결 방법:**
1. 브라우저에서 치지직에 다시 로그인
2. 확장프로그램으로 새 쿠키 복사
3. 쿠키 JSON 형식 확인:
   ```json
   {
     "NID_AUT": "실제쿠키값",
     "NID_SES": "실제쿠키값"
   }
   ```

#### 문제: 쿠키 복사가 안됨
**해결 방법:**
1. 확장프로그램이 설치되어 있는지 확인
2. 치지직 페이지에서 확장프로그램 아이콘 클릭
3. 수동으로 쿠키 입력:
   - F12 → Application → Cookies → chzzk.naver.com
   - NID_AUT, NID_SES 값 복사

### 2. 알림 관련 문제

#### 문제: 팔로워 알림이 안 뜸
**원인:**
- WebSocket 연결 실패
- 폴링 간격 설정 문제
- 네트워크 방화벽

**해결 방법:**
1. 네트워크 연결 상태 확인
2. 설정에서 갱신 주기를 5초로 변경
3. 테스트 알림(Ctrl+T)으로 동작 확인
4. 브라우저 개발자 도구에서 WebSocket 연결 상태 확인

#### 문제: 중복 알림이 계속 뜸
**원인:**
- 브라우저 캐시 문제
- 앱 재시작 없이 장시간 사용

**해결 방법:**
1. 앱 완전 종료 후 재시작
2. 브라우저 캐시 삭제 (Ctrl+Shift+Delete)
3. 설정 → 기록 전체 삭제

#### 문제: 루블리스 테스트 알림이 안됨
**해결 방법:**
1. 루블리스가 실제로 팔로우 상태인지 확인
2. 5초 쿨다운 대기
3. 앱 재시작 후 다시 시도

### 3. OBS 연동 문제

#### 문제: OBS에서 알림이 안 보임
**원인:**
- 잘못된 URL 사용
- 브라우저 소스 설정 문제
- 투명도 설정 문제

**해결 방법:**
1. 정확한 URL 사용: `http://localhost:3000/follower`
2. OBS 브라우저 소스 설정:
   - 너비: 600, 높이: 150 (가로형)
   - 너비: 300, 높이: 350 (세로형)
   - "페이지가 준비되면 새로고침" 체크
3. CSS 사용자 정의에 투명 배경 추가:
   ```css
   body { background: transparent !important; }
   ```

#### 문제: OBS에서 알림이 잘림
**해결 방법:**
1. 브라우저 소스 크기 조정
2. 설정에서 알림 레이아웃 변경 (세로형 ↔ 가로형)
3. 텍스트 크기 조정 (80% ~ 120%)

### 4. 성능 관련 문제

#### 문제: 앱이 느려짐
**원인:**
- 메모리 누수
- 과도한 로그 출력
- 네트워크 지연

**해결 방법:**
1. 앱 재시작
2. 갱신 주기를 15초 이상으로 설정
3. 히스토리 기록 삭제
4. 작업 관리자에서 메모리 사용량 확인

#### 문제: CPU 사용률이 높음
**해결 방법:**
1. 갱신 주기를 30초로 증가
2. TTS 기능 비활성화
3. 애니메이션 효과를 "페이드"로 변경
4. 백그라운드에서 실행 중인 다른 앱 종료

### 5. 네트워크 관련 문제

#### 문제: "네트워크 연결을 확인해주세요" 오류
**해결 방법:**
1. 인터넷 연결 상태 확인
2. 방화벽에서 Fazzk 허용
3. VPN 사용 시 한국 서버로 연결
4. DNS 설정 확인 (8.8.8.8, 1.1.1.1)

#### 문제: WebSocket 연결이 자주 끊어짐
**해결 방법:**
1. 네트워크 안정성 확인
2. 라우터 재시작
3. 유선 연결 사용 권장
4. 폴링 모드로 전환 (WebSocket 비활성화)

### 6. 업데이트 관련 문제

#### 문제: 자동 업데이트가 안됨
**해결 방법:**
1. 관리자 권한으로 실행
2. 바이러스 백신 프로그램 예외 처리
3. 수동 업데이트:
   - GitHub Releases에서 최신 버전 다운로드
   - 기존 앱 종료 후 새 버전 설치

#### 문제: 업데이트 후 설정이 초기화됨
**해결 방법:**
1. 설정을 다시 입력
2. 향후 업데이트를 위해 설정 백업:
   - 설정 → 내보내기 (향후 기능)

### 7. 오디오 관련 문제

#### 문제: 알림음이 안 들림
**해결 방법:**
1. 시스템 볼륨 확인
2. 앱 설정에서 볼륨 조정
3. 다른 오디오 파일로 테스트
4. 오디오 드라이버 업데이트

#### 문제: TTS 음성이 이상함
**해결 방법:**
1. Windows 음성 설정 확인
2. 한국어 TTS 엔진 설치
3. TTS 속도 조정
4. 다른 음성으로 변경

## 디버깅 방법

### 1. 로그 확인
**브라우저 개발자 도구:**
1. F12 키 또는 우클릭 → 검사
2. Console 탭에서 에러 메시지 확인
3. Network 탭에서 API 호출 상태 확인

**Tauri 앱 로그:**
1. 터미널에서 앱 실행: `npm run tauri dev`
2. 백엔드 로그 실시간 확인

### 2. 네트워크 상태 확인
```javascript
// 브라우저 콘솔에서 실행
fetch('http://localhost:3000/followers')
  .then(r => r.json())
  .then(console.log)
  .catch(console.error);
```

### 3. WebSocket 연결 테스트
```javascript
// 브라우저 콘솔에서 실행
const ws = new WebSocket('ws://localhost:3000/ws');
ws.onopen = () => console.log('WebSocket 연결됨');
ws.onerror = (e) => console.error('WebSocket 오류:', e);
ws.onmessage = (e) => console.log('메시지:', e.data);
```

## 성능 최적화 팁

### 1. 설정 최적화
- **갱신 주기**: 15-30초 (기본값: 15초)
- **표시 시간**: 3-5초 (기본값: 5초)
- **애니메이션**: 페이드 (가장 가벼움)
- **TTS**: 필요시에만 활성화

### 2. 시스템 최적화
- 백그라운드 앱 최소화
- 충분한 RAM 확보 (최소 4GB)
- SSD 사용 권장
- 정기적인 시스템 재시작

### 3. 네트워크 최적화
- 유선 연결 사용
- 안정적인 인터넷 환경
- VPN 사용 시 한국 서버 선택

## 자주 묻는 질문 (FAQ)

### Q: 여러 스트리머를 동시에 모니터링할 수 있나요?
A: 현재 버전에서는 한 번에 하나의 스트리머만 모니터링 가능합니다. 향후 업데이트에서 다중 스트리머 지원 예정입니다.

### Q: 모바일에서 사용할 수 있나요?
A: 현재는 데스크톱 전용입니다. 모바일 버전은 향후 개발 계획에 있습니다.

### Q: 다른 스트리밍 플랫폼도 지원하나요?
A: 현재는 치지직만 지원합니다. 다른 플랫폼 지원은 사용자 요청에 따라 검토하겠습니다.

### Q: 개인정보는 안전한가요?
A: 모든 데이터는 로컬에서만 처리되며, 외부 서버로 전송되지 않습니다. 쿠키 정보는 암호화하여 저장됩니다.

### Q: 상업적 사용이 가능한가요?
A: 개인 사용 목적으로 제작되었습니다. 상업적 사용에 대해서는 별도 문의 바랍니다.
