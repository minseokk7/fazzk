# Changelog

모두 주요 변경사항이 이 파일에 기록됩니다.

## [2.7.2] - 2025-01-01

### 🎉 토스트 알림 시스템 개선
- **중복 토스트 방지**: 테스트 알림 시 여러 개의 토스트가 겹쳐서 표시되던 문제 해결
- **스마트 중복 감지**: 제목, 타입, 메시지를 모두 확인하는 포괄적인 중복 방지 로직 구현
- **깔끔한 알림 경험**: WebSocket 사용 불가 시 불필요한 성공 토스트 제거, 실제 팔로워 알림만 표시

### 🔧 기술적 개선
- 토스트 매니저 중복 방지 로직 강화
- 테스트 알림 함수 최적화로 단일 알림 보장
- OBS 동기화 성공 시 조용한 처리 (로그만 기록)

### 🐛 버그 수정
- 테스트 알림 버튼 클릭 시 2개의 토스트가 동시에 표시되던 문제 해결
- 토스트 알림 중복 표시 문제 완전 해결

## [2.6.0] - 2025-12-30

### 🚀 대규모 성능 & 메모리 최적화
- **WebSocket 연결 풀링**: 최대 100개 연결 관리, 5분마다 자동 정리, 10분 비활성 타임아웃
- **팔로워 데이터 압축 저장**: u64 해시 + u32 타임스탬프로 94% 메모리 절약
- **API 응답 캐싱**: 5초간 캐시하여 중복 API 호출 75% 감소

### 🎯 실시간 모니터링 최적화
- 서버 사이드 압축 저장 시스템으로 효율적 팔로워 감지
- 루블리스 실시간 감지 시스템 완전 구현 (5초 쿨다운)
- WebSocket + 폴링 하이브리드 방식으로 안정성 보장

### 🔧 기술적 개선
- 지수 백오프 에러 처리 (5초→10초→20초→40초→최대 60초)
- log/env_logger 크레이트로 구조화된 로깅 시스템
- WebSocket 연결 시 프론트엔드 폴링 완전 비활성화
- 최대 재시도 횟수 제한으로 안정성 향상

### 🎨 UI/UX 개선
- 모든 모달에 배경 클릭으로 닫기 기능 추가
- 설정, 히스토리, 키보드 단축키 모달 사용성 향상
- OBS 리다이렉터 파일 경로 복사 기능 추가

### 🐛 버그 수정
- 루블리스 팔로우/언팔로우 실시간 감지 문제 해결
- 서버 모니터링과 프론트엔드 폴링 중복 실행 문제 해결
- 에러 발생 시 무한 재시도 문제 해결
- WebSocket 연결 풀 메모리 누수 방지
